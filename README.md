# Modbus OPC UA Server

Простой и легковесный OPC UA сервер, который транслирует данные из устройств по протоколу **Modbus over TCP** в пространство адресов **OPC UA**.

Сервер предназначен для интеграции промышленного оборудования (ПЛК, измерительные приборы) в современные SCADA-системы и MES-решения, поддерживающие стандарт OPC UA.

---

## 📋 Основные возможности

- **Поддержка Modbus TCP**: Чтение данных с удаленных устройств по промышленному протоколу.
- **OPC UA Интерфейс**: Предоставляет древовидную структуру тегов (объектов и переменных) для клиентов OPC UA.
- **Регистры хранения (Holding Registers)**: Реализована поддержка чтения 4-х регистров (Holding Registers, функция `0x03`).
- **Гибкая конфигурация**: Все настройки (адреса устройств, диапазоны регистров, имена тегов) задаются в едином Python-файле конфигурации.
- **Автономная работа**: Сервер автоматически опрашивает Modbus устройства с заданным интервалом и обновляет значения OPC узлов.

## 🚀 Быстрый старт

### Требования

- Python 3.8 или выше
- Pip (менеджер пакетов)

### Установка

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/yourusername/modbus-opcua-server.git
    cd OPC

2.  Установите зависимости:
    ```bash
    pip install -r requirements.txt

3.  Перед запуском отредактируйте файл конфигурации app/config.py
4.  Запуск
    ```bash
    python app/main_async.py

## 🐳 Запуск через Docker

### Сборка образа

1.  Из корневой директории проекта выполните команду сборки Docker образа:
    ```bash
    docker build -t opc:v1 .

2.  Запуск контейнера:
    ```bash
    docker run -d \
    --name modbus-opc \
    -e MODBUS_HOST01=host.docker.internal \
    -e MODBUS_PORT01=4001 \
    -e MODBUS_HOST02=host.docker.internal \
    -e MODBUS_PORT02=4002 \
    -p 4002:4002 \
    -p 54000:54000 \
    opc:v1
